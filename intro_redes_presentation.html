🤔 O que é uma Rede de Computadores?

Conjunto de máquinas independentes conectadas para troca de dados e recursos
Comunicação via protocolos padronizados
Compartilhamento de informações e serviços

📖 Definições Acadêmicas:

Tanenbaum (2003): Máquinas com conexão operacional via processo tecnológico
Amazon: Dispositivos interconectados que compartilham dados seguindo protocolos

📞 Analogia: Comunicação Humana

Emissor: Quem fala (computador origem)
Receptor: Quem escuta (computador destino)
Meio: Ar, telefone (cabos, WiFi)
Linguagem: Português, inglês (protocolos)
Computador A ←→ Meio de Comunicação ←→ Computador B
Usando protocolos para "conversar"

🏗️ Arquitetura Cliente-Servidor

Nós da rede divididos em clientes e servidores
Papéis fixos e bem definidos
Servidores fornecem recursos, clientes consomem

🖥️ Servidores

Fornecem serviços
Processamento poderoso
Sempre disponíveis
Gerenciam recursos
Controlam segurança

💻 Clientes

Solicitam serviços
Interface com usuário
Conectam quando necessário
Processamento local
Dependem do servidor

🌐 Exemplos Práticos

Web: Navegador (cliente) ↔ Servidor web
E-mail: Outlook (cliente) ↔ Servidor de e-mail
Jogos: Game (cliente) ↔ Servidor de jogos
Banco: App (cliente) ↔ Servidor bancário
Cliente 1 ⬋
Cliente 2 ⬌ 🖥️ SERVIDOR ⬌ Cliente 4
Cliente 3 ⬊

🔄 Arquitetura Ponto-a-Ponto (P2P)

Todos os dispositivos têm as mesmas capacidades
Cada nó atua como cliente E servidor
Sem servidor central
Distribuição de recursos e responsabilidades

✅ Vantagens

Não há ponto único de falha
Maior tolerância a falhas
Compartilhamento direto
Economia de infraestrutura
Escalabilidade natural

⚠️ Desvantagens

Difícil controle de segurança
Gerenciamento complexo
Dependência dos peers
Inconsistência de dados
Problemas legais potenciais

📁 Exemplos de P2P

BitTorrent: Compartilhamento de arquivos
Bitcoin: Moeda digital descentralizada
Skype: Chamadas diretas entre usuários
Blockchain: Banco de dados distribuído
Peer A ⬌ Peer B
⬍ ⬌ ⬌ ⬎
Peer D ⬌ Peer C
(Todos conectados entre si)

🔌 Topologia Física vs Lógica

📐 Topologia Física

Como os dispositivos estão fisicamente conectados
Layout real dos cabos e equipamentos
Determina custo de instalação
Influencia na manutenção

💡 Topologia Lógica

Como os dados trafegam na rede
Caminho dos dados independe do físico
Definida por protocolos
Pode ser diferente da física

🎯 Exemplo Prático

Uma rede pode ter topologia física em estrela (todos conectados a um switch)
Mas topologia lógica de barramento (dados vão para todos)
O importante é entender que são conceitos diferentes!

🏢 Analogia: Prédio de Escritórios

Física: Salas conectadas por corredores
Lógica: Como as pessoas se comunicam (e-mail, telefone, pessoalmente)

📐 Tipos de Topologia Física

⭐ Estrela

Dispositivos conectados a um ponto central
Falha no centro para toda rede
Fácil manutenção
Mais cabos necessários

🔄 Anel

Dispositivos em forma de círculo
Dados circulam em uma direção
Falha em um nó afeta todos
Performance previsível

🚌 Barramento

Cabo principal com derivações
Compartilhamento do meio
Economia de cabos
Dificuldade de manutenção

🕸️ Malha

Múltiplas conexões entre nós
Alta tolerância a falhas
Custo elevado
Redundância de caminhos

🏆 Mais Comum Hoje

Topologia estrela é a mais usada em redes modernas
(WiFi doméstico, switches empresariais)

💡 Topologia Lógica - Como os Dados Trafegam

📡 Ethernet (Broadcast)

Dados enviados para todos
Só o destinatário processa
Outros ignoram a mensagem
Padrão mais comum hoje
Analogia:
Como gritar em uma sala:
todos ouvem, mas só quem
foi chamado responde

🎫 Token Ring

Token (ficha) circula na rede
Só quem tem o token transmite
Acesso controlado e ordenado
Menos usado atualmente
Analogia:
Como usar um bastão para
falar em uma reunião:
só quem tem o bastão fala
Exemplo Ethernet:
Computador A quer enviar para B
→ A envia para todos (A, B, C, D)
→ Todos recebem, mas só B processa
→ C e D ignoram a mensagem

🌐 Na Prática

Sua rede WiFi doméstica usa Ethernet (broadcast)
mesmo tendo topologia física em estrela!

🏠 LAN - Local Area Network

🏠 Rede Local

Pequena área geográfica - casa, escritório, prédio

✅ Características

Alta velocidade (100 Mbps a 10 Gbps)
Baixa latência
Propriedade única
Controle total sobre a rede
Custo baixo de operação

🎯 Aplicações

Compartilhamento de arquivos
Impressão em rede
Jogos locais
Acesso à Internet compartilhado
Sistemas internos da empresa

🏠 Exemplos de LAN

Residencial: WiFi doméstico conectando celulares, notebooks, smart TV
Escritório: Computadores conectados por switches
Escola: Laboratório de informática
Café: WiFi para clientes

📏 Tamanho Típico

De alguns metros até poucos quilômetros
Velocidades muito altas, controle local

🏙️ MAN - Metropolitan Area Network

🏙️ Rede Metropolitana

Área de uma cidade ou região metropolitana

✅ Características

Velocidade moderada a alta
Abrange uma cidade
Conecta várias LANs
Pode ser pública ou privada
Uso de fibra óptica comum

🎯 Aplicações

Campus universitário distribuído
Filiais de uma empresa na cidade
Rede municipal de governo
Provedores de Internet locais
TV a cabo e telefonia

🏫 Exemplos de MAN

Universidade: Campus principal conectado com campi satélites
Governo: Prefeitura conectada com escolas e postos de saúde
Empresa: Matriz conectada com filiais na mesma cidade
Provedor: ISP conectando bairros de uma cidade

📏 Tamanho Típico

De alguns quilômetros até 100 km
Velocidades boas, controle regional

🌍 WAN - Wide Area Network

🌍 Rede de Longa Distância

Cobre países, continentes ou o mundo todo

✅ Características

Velocidade variável
Latência maior
Conecta MANs e LANs
Uso de infraestrutura pública
Custo elevado

🎯 Tecnologias

Cabos submarinos
Satélites
Fibra óptica transcontinental
Redes de telefonia
Internet pública

🌐 Exemplos de WAN

Internet: A maior WAN do mundo!
Multinacional: Matriz no Brasil conectada com filial no Japão
Governo: Rede federal conectando todos os estados
Banco: Agências de todo o país conectadas

📏 Tamanho Típico

Centenas ou milhares de quilômetros
Velocidades menores, alcance global

LAN (casa) → MAN (cidade) → WAN (mundo) → Internet
