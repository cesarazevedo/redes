📜 História: O Problema Original

🏭 Redes Proprietárias

Cada fabricante tinha sua própria tecnologia
Impossível misturar soluções diferentes
Falta de compatibilidade
Fabricante controlava todos os componentes

⚠️ Problemas Resultantes

Cliente "preso" a um fornecedor
Custos altos de expansão
Dificuldade de integração
Evolução tecnológica limitada

💡 A Solução da ISO

International Standards Organization
Modelo OSI (Open Systems Interconnection)
Padrão para interconexão de sistemas
7 camadas bem definidas
Fabricantes criam componentes compatíveis
Antes: Cada fabricante = ilha isolada
Depois: Padrão OSI = comunicação universal

🔌 Camada 1 - Física

Características físicas, elétricas e ópticas
Totalmente ligada ao hardware
Especifica conectores e cabos
Transparente para o usuário

Cabos e conectores

Conversores de mídia
RJ-11
RJ-45
BNC
Fibra

⚡ Funções Principais

Transmitir bits (0s e 1s)
Converter sinais elétricos/ópticos
Definir tensões e temporização
Especificar conectores

📞 Analogia do Telefone

Como os fios e tomadas do telefone - conectam fisicamente, mas não sabem o que está sendo falado.

💡 Exemplo Prático

Não sabe se é e-mail, web ou arquivo

🔗 Camada 2 - Enlace de Dados

Interface confiável entre físico e dados
Detecta erros e controla fluxo
Organiza bits em quadros (frames)
Endereçamento local (MAC)

🎯 Funções Principais

Enquadramento (framing)
Detecção de erros (CRC)
Controle de fluxo local
Endereçamento MAC

🔧 Protocolos Comuns

WiFi 802.11
PPP

📡 Subcamadas IEEE

LLC: Logical Link Control
MAC: Media Access Control

🏷️ Endereço MAC

00:1B:63:84:45:E6
Identifica cada placa
globalmente única

📞 Analogia do Telefone

Como tirar o fone do gancho e ouvir o tom - verifica se a linha está funcionando antes de discar.

Exemplo: Se houver ruído na linha, esta camada pede para repetir. É como dizer "Não ouvi, pode repetir?"

💡 Moral da História

Cada camada tem sua responsabilidade específica
Trabalham juntas para realizar a comunicação
Usuário só vê a camada 7, mas todas são essenciais

📦 Encapsulamento de Dados

Cada camada adiciona suas informações
Como "bonecas russas" ou "caixas dentro de caixas"
Descapsulamento no destino
Processo transparente ao usuário
Envio (Encapsulamento):

Dados do usuário
↓ +Cabeçalho Aplicação
↓ +Cabeçalho Apresentação
↓ +Cabeçalho Sessão
↓ +Cabeçalho Transporte
↓ +Cabeçalho Rede
↓ +Cabeçalho Enlace
↓ Sinais Físicos

Recepção (Descapsulamento):
Processo reverso remove cada cabeçalho

📮 Analogia do Correio

Carta = dados do usuário
Envelope = cabeçalhos das camadas
Endereço = informações de roteamento
Caminhão dos correios = meio físico

🔍 Por que Encapsular?

Cada camada precisa de suas informações
Roteamento, controle, endereçamento
Permite funcionamento independente
Facilita manutenção e evolução
