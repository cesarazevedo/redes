🤔 SSH vs RDP: Qual Escolher?

🔧 SSH 🖥️ RDP
O que é Terminal texto (linha de comando) Desktop gráfico completo
Melhor para Servidores Linux, comandos, scripts Computadores Windows, uso normal
Velocidade Muito rápido (só texto) Mais lento (gráficos)
Segurança Excelente (criptografia forte) Boa (criptografia do Windows)
Uso de dados Mínimo Alto

💡 Dica Simples:

SSH: Para mexer em servidores como um "hacker do bem"
RDP: Para usar outro Windows como se fosse seu

🔐 Como Funciona a Segurança?

🔑 SSH

Criptografia super forte
Pode usar senha ou chaves
Chaves são como "impressão digital digital"
Impossível de interceptar

🖥️ RDP

Criptografia do Windows
Usa senhas do usuário Windows
Pode usar certificados
Seguro para redes locais

🏰 Analogia da Fortaleza:

Imagine que cada computador é uma fortaleza. SSH e RDP são túneis seguros que você escava para entrar. A diferença é que SSH te leva para a "sala de comando" (terminal) e RDP te teletransporta diretamente para dentro da fortaleza (desktop).

🏗️ Como os Dois Funcionam

SSH (PuTTY) [Seu PC] ←→ [PuTTY] ←→ [Servidor Linux/Unix] Terminal Comandos de texto RDP (Remote Desktop) [Seu PC] ←→ [Remote Desktop] ←→ [PC Windows] Tela gráfica Desktop completo

☁️ Lado Servidor

🚀 PRÁTICA: Baixando o PuTTY (Versão Portable)

PuTTY é o cliente SSH mais popular no Windows - gratuito, confiável e sem necessidade de instalação.

Passo a Passo:

1. Vá para putty.org e role até "Alternative binary files".
2. Em "putty.exe", baixe a versão "64-bit x86".
3. Salve o arquivo na pasta Downloads ou crie uma pasta "PuTTY".
4. Não precisa instalar! É só clicar duas vezes no arquivo .exe para executar.

✨ Vantagens da Versão Portable:

Sem instalação: Executa direto do arquivo
Portável: Pode copiar para pendrive
Sem alterações no sistema: Não modifica registro do Windows

📁 Opcional: Organize melhor

Crie uma pasta C:\Ferramentas\PuTTY\ e coloque o arquivo lá. Assim fica mais fácil de encontrar!

🚀 PRÁTICA: Primeira Conexão SSH

Vamos conectar em um servidor usando PuTTY.

Passo a Passo:

1. Execute o putty.exe (clique duas vezes no arquivo baixado).
2. Em "Host Name", digite o IP ou nome do servidor (ex: 192.168.1.100).
3. Deixe "Port" como 22 e "Connection type" como SSH.
4. Clique em "Open".

⚠️ Primeira Vez: Verificação de Segurança

O PuTTY mostrará um alerta sobre a "chave do host". É normal! Clique em "Accept" para confiar no servidor.

PuTTY Security Alert The host key is not cached... Do you trust this host? [Yes] [No] [Cancel]

Agora: Digite seu usuário e senha quando solicitado. Você estará dentro do servidor!

🚀 PRÁTICA: Criando Chaves SSH com PuTTYgen

Chaves SSH são mais seguras que senhas. Vamos criar sua "impressão digital digital".

💾 Primeiro: Baixe o PuTTYgen

Vá em putty.org > "Alternative binary files" > puttygen.exe (64-bit x86). Salve na mesma pasta do PuTTY.

Passo a Passo:

1. Execute o puttygen.exe (clique duas vezes).
2. Escolha "RSA" e 2048 bits (padrão).
3. Clique em "Generate".
4. Mova o mouse aleatoriamente sobre a área branca para gerar aleatoriedade.
5. Digite um comentário (ex: "Minha chave SSH").
6. Opcionalmente, digite uma senha para proteger a chave.
7. Clique em "Save private key" e salve como minha_chave.ppk.
8. Copie o texto da "Public key" (a chave pública).

Resultado: Você tem um arquivo .ppk (chave privada) e uma chave pública (texto copiado).

🚀 PRÁTICA: Instalando sua Chave no Servidor

Agora vamos "autorizar" sua chave no servidor para login sem senha.

Método Simples:

1. Conecte no servidor via PuTTY (com senha, como antes).
2. Digite: mkdir -p ~/.ssh e pressione Enter.
3. Digite: nano ~/.ssh/authorized_keys e pressione Enter.
4. Cole sua chave pública (botão direito do mouse no PuTTY).
5. Pressione Ctrl+X, depois Y, depois Enter para salvar.
6. Digite: chmod 600 ~/.ssh/authorized_keys para definir permissões.

🎯 Teste:

Feche o PuTTY, abra novamente, mas agora em "Connection > SSH > Auth > Credentials", clique em "Browse" e selecione seu arquivo .ppk. Conecte - não pedirá mais senha!

🚀 PRÁTICA: Transferindo Arquivos

Para transferir arquivos via SSH, temos duas opções principais.

💎 Opção 1: WinSCP (Recomendado)

1. Baixe o WinSCP em winscp.net (versão portable).
2. Execute o WinSCP.exe.
3. Digite IP do servidor e usuário.
4. Em "Advanced" > "SSH" > "Authentication", selecione seu arquivo .ppk.
5. Clique "Login".
6. Arraste e solte arquivos entre as janelas!

⚡ Opção 2: SCP no Terminal

Use o PowerShell do Windows (mais avançado)
# Enviar arquivo scp C:\arquivo.txt user@servidor:/home/ # Baixar arquivo scp user@servidor:/arquivo.txt C:\Downloads\

Nota: Precisa converter a chave .ppk para OpenSSH primeiro.

💡 Dica:

O WinSCP é mais fácil para iniciantes - interface gráfica como Windows Explorer!

🚀 PRÁTICA: Habilitando RDP no Windows

RDP permite controlar outro PC Windows como se você estivesse na frente dele.

No PC que você quer acessar remotamente:

1. Clique com botão direito em "Este Computador" > "Propriedades".
2. Clique em "Configurações avançadas do sistema".
3. Na aba "Remoto", marque "Habilitar Área de Trabalho Remota".
4. Clique em "OK".

⚠️ Importante para Segurança:

Anote o nome do computador (aparece na tela)
Certifique-se de que o usuário tem senha forte
Considere adicionar o usuário ao grupo "Usuários da Área de Trabalho Remota"

🚀 PRÁTICA: Conectando via Remote Desktop

Agora vamos conectar do seu PC para o PC que habilitou RDP.

Passo a Passo:

1. No seu PC, pressione Windows + R.
2. Digite mstsc e pressione Enter.
3. Em "Computador", digite o nome do PC ou IP (ex: DESKTOP-ABC123 ou 192.168.1.50).
4. Digite seu usuário e senha.
5. Clique em "Conectar".

🎉 Resultado:

Uma nova janela abrirá mostrando o desktop do outro computador! Você pode usar normalmente como se estivesse sentado na frente dele.

Dica: Para sair, simplesmente feche a janela ou clique no X no topo da barra de conexão.

🚀 PRÁTICA: RDP pela Internet (Avançado)

Para acessar de qualquer lugar, precisa configurar o roteador.

Configuração no Roteador:

1. Acesse seu roteador (geralmente 192.168.1.1 no navegador).
2. Procure por "Port Forwarding" ou "Redirecionamento de Porta".
3. Crie uma regra: Porta 3389 (RDP) → IP do seu PC.
4. Descubra seu IP público em whatismyip.com.

⚠️ Cuidados de Segurança:

Mude a porta padrão (3389 → outra, como 3390)
Use senhas muito fortes
Considere usar VPN em vez de RDP direto
Monitore tentativas de login suspeitas

Para conectar: Use o IP público + porta (ex: 203.45.67.89:3390)

🔧 Resolvendo Problemas Comuns

🔧 Problemas SSH/PuTTY:

Connection refused: Servidor SSH não está rodando
Permission denied: Usuário/senha incorretos
Key não funciona: Verifique se está no arquivo authorized_keys
Timeout: Firewall bloqueando porta 22

🖥️ Problemas RDP:

RDP não habilitado: Verifique se está ativado
Usuário sem permissão: Adicione ao grupo RDP
Firewall: Libere porta 3389
"Usuário logado": Só um usuário por vez no Windows Home

🛠️ Comandos Úteis para Diagnóstico:

Testar conexão: telnet IP 22 (SSH) ou telnet IP 3389 (RDP)
Ver logs: Visualizador de Eventos do Windows
IP do PC: ipconfig no cmd

SSH + RDP: Acesso total a qualquer computador, de qualquer lugar.

mstsc para conectar
Habilitar no PC de destino
Port forwarding para internet
Ideal para PCs Windows
Interface gráfica completa
